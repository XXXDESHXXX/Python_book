База данных - набор взаимосвязанных данных.
Система управления базами данных (СУБД) - комплекс программных средств для управления данными.
Отвечает за: поддержку языка БД, механизмы хранения и извлечения данных, и т.д.
БД делятся на:
Файл-серверные, клиент-серверные, встраиваемые.
Для реляционных СУБД теоретической основой служит реляционная алгебра.
Реляционная модель:
Сущность - клиенты, заказы и т.д.
Таблица - отношение
Столбец - атрибут
Строка/запись - кортеж
SQL (Structured Query Language) - язык запросов, который применяют, чтобы работать с базами данных, структурированных особым образом
Результатом SQL запроса является результирующий набор(таблица)
SQL категоризируется на:
DDL (Data Definition Language) - CREATE, ALTER, DROP.
DML (Data Manipulation Language) - SELECT, INSERT, UPDATE, DELETE.
TCL (Transaction Control Language) - COMMIT, ROLLBACK, SAVEPOINT.
DCL (Data Control Language) - GRANT, REVOKE, DENY.
Типы данных в PostgreSQL:
- Integral Numbers:
smallint, integer, bigint
- Real Numbers:
 decimal/numeric, real/float4, double precision/float8/float.
 - Integral Numbers(автоинкрементирование):j
 smallserial, serial, bigserial
 - Characters:
 char, varchar, text
 - Logical:
Boolean, bool
- Temporal:
date, time, timestamp, interval, timestamptz
Отношение между таблицами в SQL:
Один ко многим - вид отношения, при котором один элемент из таблицы a может относиться к нескольким из таблицы b.
Один к одному - вид отношения, при котором один элемент из табилцы a относиться к одному элементу из таблицы b.
Многие ко многим - вид отношения, при котором несколько элементов из таблицы a относяться к нескольким элементам из таблицы b.
